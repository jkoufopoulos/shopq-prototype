System Architecture

5 beats: capture ‚Üí classify ‚Üí learn ‚Üí narrate ‚Üí delight

1
1. CLIENT (Capture)
Chrome extension scans Gmail for unlabeled threads. Checks cache, runs pattern detectors, batches requests.
2
2. GATEWAY (Route)
FastAPI handles auth, rate limiting, and routes to /api/organize or /api/context-digest.
3
3. CLASSIFY (Pipeline)
Multi-stage: Rules ‚Üí Memory ‚Üí LLM (gemini-2.0-flash) ‚Üí Verifier. Rules match 50-70% at $0 cost.
4
4. ENRICH (Temporal)
Add urgency signals, apply temporal decay. Near-term events boosted, distant events decay.
5
5. PERSIST (Database)
Store in mailq.db: 13 tables including rules, feedback, confidence_logs, email_threads.
6
6. NARRATE (Digest)
V2 pipeline: Extract entities ‚Üí Importance ‚Üí Temporal ‚Üí Sections ‚Üí Narrative ‚Üí Render ‚Üí Verify.
7
7. DELIVER (Feedback)
Send digest via Gmail API. User feedback creates rules with 0.95 confidence.
