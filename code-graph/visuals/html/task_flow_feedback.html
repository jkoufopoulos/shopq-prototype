Feedback Learning Loop

How user corrections become rules

1
User Corrects Label
User manually changes a label in Gmail (e.g., Critical ‚Üí Routine).
2
Detect Change
Content script detects label change via MutationObserver watching Gmail DOM.
3
Send Feedback
POST to /api/feedback with thread_id, old_label, new_label. File: feedback.py
4
Log Feedback
Store correction in feedback table for pattern analysis. File: feedback_learning.py
5
Pattern Check
Check if 3+ similar corrections exist (same sender/domain pattern).
6
Create Rule
If pattern found, create rule with 0.95 confidence. INSERT INTO rules table. File: rules_engine.py
7
Silent Confirm
Return success to extension. No UI change - user doesn't see feedback was captured.
